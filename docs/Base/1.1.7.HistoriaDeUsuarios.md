## Critérios de Aceitação

#### US01 (RF14 - Autenticação)
* **Cadastro:** O sistema deve exigir e-mail, nome de usuário e senha. A senha deve ser criptografada antes de ir para o banco de dados. E-mails duplicados não devem ser permitidos.
* **Login:** O acesso só deve ser liberado se o e-mail e a senha baterem exatamente com o banco de dados. Em caso de erro, exibir aviso ao usuário.
* **Recuperação:** O sistema deve ter um fluxo básico para permitir a troca de uma senha esquecida via e-mail ou SMS.

#### US02 (RF07 - Criar Tópicos)
* **Bloqueio de Visitantes:** Se um usuário não logado tentar criar um tópico, deve ser redirecionado para a tela de login.
* **Validação:** Os campos "Título" e "Conteúdo" são obrigatórios. O botão de enviar deve ficar desabilitado ou retornar erro se um deles estiver vazio.
* **Ação Final:** Ao salvar no banco de dados com sucesso, o usuário deve ser levado direto para a tela do tópico recém-criado.
* **Responsividade:** O formulário e o botão de envio devem caber perfeitamente na tela de um celular, sem barra de rolagem horizontal.

#### US03 (RF08 - Comentar em Tópicos)
* **Bloqueio de Visitantes:** Apenas usuários com sessão ativa podem enviar comentários.
* **Validação:** Comentários vazios devem ser rejeitados pelo sistema.
* **Visualização:** Assim que o comentário é salvo no banco, ele deve aparecer imediatamente no final da lista de comentários do tópico, mostrando o nome do autor e a data/hora.

#### US04 (RF01 - Ver Discussões Recentes)
* **Consulta e Ordem:** O sistema deve buscar os dados no banco e carregar os tópicos ordenados por data de criação de forma decrescente.
* **Dados Exibidos:** Cada item da lista deve mostrar obrigatoriamente: Título do tópico, autor que criou e a data/hora da criação.
* **Responsividade:** A visualização do feed deve se adaptar a telas de celulares, quebrando textos longos para a linha de baixo se necessário, sem deformar o layout.
