# 1.3.2 XP

A modelagem a seguir utiliza a notação BPMN (Business Process Model and Notation) para representar o fluxo de trabalho da metodologia XP, focando na excelência técnica e na agilidade do desenvolvimento de software. O objetivo deste diagrama é ilustrar como as práticas de engenharia são integradas ao ciclo de vida do projeto, garantindo que o código produzido seja robusto, testado e desenvolvido de forma colaborativa.

O fluxo é estruturado em fases temporais que organizam as atividades desde o planejamento até a entrega do incremento, evidenciando o papel central do Time de Desenvolvimento na execução das práticas ágeis:

* **Início e Planejamento:** O processo se inicia com o Sprint Planning, onde os requisitos priorizados são traduzidos em tarefas técnicas prontas para a iteração.
* **Validação e Aceitação:** Antes do desenvolvimento massivo, o fluxo passa pelos Testes de Aceitação. Um gateway exclusivo verifica se os critérios foram aceitos; em caso negativo, o fluxo retorna ao planejamento para ajustes imediatos.
* **Ciclo de Desenvolvimento (Core XP):** Esta fase representa o coração da metodologia, onde o desenvolvimento é sustentado pela principal prática, que consiste em programar em pares (**Pair Programming**): Onde dois desenvolvedores trabalham na mesma estação para maximizar a qualidade e a troca de conhecimento.
* **Deploy e Continuidade:** Após a conclusão do ciclo de desenvolvimento, o processo avança para a Revisão da Iteração e Retrospectiva, onde a qualidade do código e a dinâmica da equipe são avaliadas antes do encerramento da fase.

Este modelo reforça a mentalidade de feedback constante da XP, utilizando gateways de decisão para assegurar que nenhuma funcionalidade avance para a próxima etapa sem atingir os padrões de qualidade definidos pela equipe.
